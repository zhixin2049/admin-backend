# 2026-05-21 问题与解决方法汇总

> 范围：前台注册/登录页面 + 管理后台用户管理（注册用户、管理员、角色权限）
> 项目：React-admin（React 18 + TypeScript + Ant Design 6.4.3 + Zustand）

---

## 一、前台注册/登录页面

### 1. 未注册账号也能登录

- **问题**：输入一个不存在的账号名，系统也能"登录成功"
- **根因**：`LoginPage.tsx` 的 `handleSubmit` 只做了非空校验（`if (!account \|\| !password)`），从未实际验证账号是否在系统中存在、密码是否匹配
- **关联根因**：`memberApi.create()` 注册时接收了密码参数但从未存储——`Member` 类型根本没有 `password` 字段
- **解决方法**：
  - 新增 `mock_credentials` localStorage 键，用 `{ username: password, phone: password }` 存储密码凭证
  - `memberApi` 新增 `login(account, password)` 方法，依次检查：账号存在 → 未被禁用 → 密码匹配
  - `LoginPage.tsx` 改为 async，增加 `submitting` / `submitError` 状态，按钮显示"登录中..."，错误信息精确提示
- **涉及文件**：`src/api/index.ts`、`src/pages/home/LoginPage.tsx`

### 2. 旧账号登录提示密码错误

- **问题**：凭证系统上线前注册的账号（mock 数据中已有的 12 条），用正确密码登录却提示"密码错误"
- **根因**：`mock_credentials` 是新增键，旧账号的密码从未被存入。`creds[username]` 返回 `undefined`，直接判定为密码错误
- **解决方法**：`memberApi.login()` 增加兼容逻辑——当 `storedPwd === undefined` 时视为旧账号首次登录，将用户输入的密码补录到 `mock_credentials`，后续正常校验
- **涉及文件**：`src/api/index.ts`

### 3. 登录成功后导航栏状态不更新

- **问题**：在 `/login` 页登录成功跳转到首页，导航栏仍然显示"登录/注册"按钮，没有切换到已登录状态
- **根因**：`LoginPage` 登录成功仅设置了本地 `loggedIn` 状态（`useState`），`Navbar` 组件完全无法感知
- **解决方法**：
  - 新建 `useUserStore`（Zustand + persist，key: `user-storage`），存储当前登录的 `Member` 对象
  - `LoginPage` 登录成功后调用 `setUser(member)` 写入全局 store
  - `Navbar` 读取 `useUserStore`：已登录 → 显示"用户名 + 退出按钮"；未登录 → 显示"登录/注册"
  - 退出时清空 store 并跳转首页
- **涉及文件**：`src/store/index.ts`、`src/pages/home/LoginPage.tsx`、`src/components/home/navbar/Navbar.tsx`

### 4. 字段命名不统一

- **问题**：登录、注册、后台管理三个地方对同一字段的标签和字段名不一致
- **解决方法**：全链路统一到以下约定：

| 场景 | 标签 | 前端字段名 | 校验规则 |
|------|------|--------|----------|
| 登录 | 账号/手机 | account | 非空 |
| 登录 | 用户密码 | password | 非空 |
| 注册 | 账号名称 | username | 6~18位数字/字母 |
| 注册 | 账号密码 | password | 6~12位含数字+字母 |
| 注册 | 手机号码 | phone | 1[3-9]\d{9} |
| 注册 | 用户性别 | gender | 1:男 2:女 |
| 注册 | 所在省份 | province | 31省下拉 |

- **涉及文件**：`LoginPage.tsx`、`RegisterPage.tsx`、`MemberList.tsx`

## 二、管理后台 — 注册用户管理

### 5. 仪表盘数据不随注册用户增长更新

- **问题**：注册了多个用户后，仪表盘仍显示"注册用户 12，今日新增 0"
- **根因**：`dashboardApi.getStats()` 直接返回静态 `mockDashboardStats`（写死的 `totalMembers: 12, todayNewMembers: 0`），从未读取真实的 localStorage 数据
- **解决方法**：`getStats()` 改为调用 `loadMembers()` 动态计算：
  - `totalMembers` = 实际注册用户总数
  - `todayNewMembers` = 今日新增数（比较 `registeredAt`）
- **涉及文件**：`src/api/index.ts`

### 6. 注册用户 — 新增/编辑时密码处理

- **问题**：后台新增用户时需要填写密码，但注册用户的 `Member` 类型没有 `password` 字段
- **解决方法**：
  - `memberApi.create()` 去重校验后，将密码单独写入 `mock_credentials`，不在 `mock_members` 中暴露
  - 编辑弹窗新增「账号密码」字段（`Input.Password`），placeholder 为"不修改密码留空"，密码为空时不修改
  - `memberApi.update()` 签名扩展支持 `password` 参数
- **涉及文件**：`src/api/index.ts`、`src/pages/user/members/MemberList.tsx`

## 三、管理后台 — 管理员账号管理

### 7. 管理员修改密码后登录失败

- **问题**：在管理后台修改了 `admin` 的密码，退出后用新密码登录，提示"账号或密码错误"
- **根因**：双问题——
  - `authApi.login()` 中密码写死 `admin123`（`if (form.username === 'admin' && form.password === 'admin123')`）
  - `adminApi.update()` 更新了 `mock_admins` 但从未保存密码
- **解决方法**：
  - 新增 `mock_admin_credentials` localStorage 键，存储 `{ admin: password, ... }`
  - `authApi.login()` 改为从 `mock_admin_credentials` 读取对应用户名的密码
  - `adminApi.update()` 修改密码时同步写入 `mock_admin_credentials`
- **涉及文件**：`src/api/index.ts`

### 8. 管理员和角色数据刷新后丢失

- **问题**：在后台添加了管理员或角色，刷新页面后新增数据全部消失
- **根因**：`adminApi` 和 `roleApi` 的 CRUD 方法只操作内存数组 `mockAdmins` / `mockRoles`，浏览器刷新后内存清空，回到 mock 初始值
- **解决方法**：
  - 新增 `mock_admins` / `mock_roles` localStorage 键
  - 新增 `loadAdmins()` / `saveAdmins()` 和 `loadRoles()` / `saveRoles()` 辅助函数
  - `adminApi` 和 `roleApi` 所有方法改为 **load → 操作 → save** 模式
  - 与已有的 `memberApi` / `navMenuApi` 持久化模式统一
- **涉及文件**：`src/api/index.ts`

### 9. 新增的管理员无法登录

- **问题**：在后台创建了一个新管理员账号，退出后用它登录，提示"账号或密码错误"
- **根因**：两个问题叠加——
  1. `authApi.login()` 硬编码只校验 `username === 'admin'`，其他管理员名字一律拒绝
  2. `adminApi.create()` 只写入了 `mock_admins`，未将密码写入 `mock_admin_credentials`
- **解决方法**：
  - `authApi.login()` 改为从 `mock_admins` 动态查找管理员 → 检查 `status` 是否禁用 → 从 `mock_admin_credentials` 按用户名读密码校验
  - `adminApi.create()` 增加去重检查 + 提取 `password` 参数写入 `mock_admin_credentials`
  - `authApi.getProfile()` 改为 `loadAdmins()[0]`，不再死绑 `mockLoginResult`
- **涉及文件**：`src/api/index.ts`

### 10. 管理后台显示管理员名称为 superadmin（非预期）

- **问题**：登录后顶栏显示管理员名是 `superadmin`，但实际登录的是 `admin`
- **根因**：`mock/index.ts` 中初始管理员用户名为 `'superadmin'`，不是 `'admin'`
- **解决方法**：将 mock 数据中的管理员用户名改为 `'admin'`
- **涉及文件**：`src/mock/index.ts`

## 四、管理后台 — 角色与权限

### 11. 左侧菜单不按角色权限过滤

- **问题**：不管用什么角色登录，左侧菜单都显示全部（包括管理员账号、角色权限、系统设置等）
- **根因**：`SideMenu.tsx` 渲染时直接把整个 `menuRoutes` 传给 antd Menu，从未过滤
- **解决方法**：
  - 新增 `canAccess()` 函数：`meta.roles` 未设置 → 所有人可见；已设置 → 检查 `admin.role` 是否在列表中（superadmin 通吃）
  - 新增 `filterRoutes()` 递归过滤路由树，父菜单至少有一个子菜单可见才显示
  - `adminApi.create()` / `adminApi.update()` 增加 `roleId` → `role` slug + `roleName` 自动映射
- **涉及文件**：`src/components/layout/SideMenu.tsx`、`src/api/index.ts`

### 12. 自定义角色分配的权限不生效

- **问题**：创建了一个自定义角色"用户运营"，只分配了"注册用户"的查看和编辑权限。但登录后左侧菜单什么都不显示
- **根因**：`canAccess()` 只用角色 **slug** 做硬匹配（如 `superadmin`、`editor`）。自定义角色 slug 如 `yonghu` 不在任何 `meta.roles` 列表中，所有路由都被过滤掉
- **解决方法**：实现双重权限检查——
  - **第 1 层**：slug 匹配（`meta.roles` 列表中的字面匹配，保持兼容）
  - **第 2 层**：resource:action 匹配——新增 `ROUTE_PERMISSION` 映射表，12 条路由映射到所需权限；从 localStorage 加载当前角色的 `permissions` 列表，逐条比对
  - 任一命中即放行，superadmin 跳过所有检查
- **涉及文件**：`src/components/layout/SideMenu.tsx`（+70 行）、`src/components/common/AuthGuard.tsx`、`src/router/index.tsx`、`src/api/index.ts`

### 13. 内容管理页面权限隔离

- **问题**：用户运营角色只应有"用户管理"的权限，但实际还能看到和访问"内容管理"（轮播图、专业分类等）
- **根因**：`routes.ts` 中内容管理子路由没有设置 `meta.roles`，`canAccess()` 判定为"无角色限制 → 所有人可见"
- **解决方法**：内容管理所有 5 个子路由加上 `roles: ['superadmin']`（轮播图管理、专业分类、专业主页、视频分组、视频列表）
- **涉及文件**：`src/router/routes.ts`

## 六、关键数据流总结

```
┌─────────────────────────────────────────────────────────────┐
│                       localStorage                          │
├──────────────────┬──────────────────┬──────────────────────┤
│ mock_members     │ mock_credentials │ mock_admin_credentials│
│ 注册用户列表     │ 前台密码凭证     │ 后台管理员密码凭证    │
│ Member[]         │ {user:pwd, ...}  │ {admin:pwd, ...}     │
├──────────────────┼──────────────────┼──────────────────────┤
│ mock_admins      │ mock_roles       │ mock_nav_menus       │
│ 管理员列表       │ 角色与权限列表   │ 导航菜单             │
│ Admin[]          │ Role[]           │ NavMenu[]            │
└──────────────────┴──────────────────┴──────────────────────┘
```

| 操作 | 写入的键 |
|------|---------|
| 前台注册 | `mock_members` + `mock_credentials` |
| 前台登录 | 读取 `mock_members` + `mock_credentials` |
| 后台登录 | 读取 `mock_admins` + `mock_admin_credentials` |
| 新增管理员 | `mock_admins` + `mock_admin_credentials` |
| 修改管理员密码 | `mock_admin_credentials` |
| 新增/编辑角色 | `mock_roles` |
| 权限检查 | 读取 `mock_roles`（permissions 列表） |
